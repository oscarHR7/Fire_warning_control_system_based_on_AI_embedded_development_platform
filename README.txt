

/*************************************************************框架性说明*************************************************************/
项目结构：搭建一个火灾预警控制系统，核心包括前端AI火焰识别装置、后端检测环境实时图像流传输及相关命令控制平台、前端警示装置三部分。
系统目的：目的就是后端对前端的远程控制，而且要保证设计的准确性和可靠性。通常火灾发生的突然性，在检测设备部署好后，必须进行24小时的全方位检测，故该系统在上电后检测模块将一直持续运行，并实时传输数据。火灾发生的检测可以采用机器学习或简单的传感器判断原理进行预处理。考虑到过程存在一定的误差性需要进行后端的人工确认后再进行相对应的决定是否触发前端的报警信号。
设计模块：STM32和89C51两种MCU，考虑到后端处理的数据量和复杂度相较于前端都要更多，综合功能需求更大，所以采用STM32，而对于前端控制电路仅仅需要作出相关展示即可，因而从成本的角度出发采用C51即可。
硬件方面：主要涉及到检测电路和展示电路以及后端控制电路。整体上可以稳定达到一定的效果要求，在信号传输过程中采用RS485有效抑制信号干扰，同时采用无线串口模块实现远距离通信。
软件方面：以STM32为控制核心部分，经过图像传输，最后通过后端处理系统控制前端具体显示结果。火焰识别方面采用Thensorflow深度学习框架。
